### Платформенная альтернатива — пересмотрено (2026-04-21, поздний вечер)

Основатель возразил: «Код пишет Claude Code (не bottleneck), бренд
SecAudit не токсичен (safety=надёжность), дизайн переделаем». Пересмотр
был сделан честно:

**Что из прошлого анализа исправлено:**
- ❌ «Multi-vertical требует команды (код-bottleneck)» — ошибочно. Claude
  Code снимает код-bottleneck (MVP вертикали 1-2 нед вместо 4-6).
- ❌ «Бренд Sec токсичен для всего» — преувеличено. Для смежных
  compliance-вертикалей (HR/юр/мед compliance) — работает. Для далёких
  (риэлтор/бух/самозанятый) — ограничивает.
- ✅ Дизайн не bottleneck (DESIGN.md + Claude Code).

**Что остаётся как реальный bottleneck (не решается Claude Code):**
- 🚨 Доменная экспертиза в каждой вертикали (проверять отсылки к законам)
- 🚨 Маркетинг ×N каналов: 2 канала = 12-20ч/нед, 3+ = выход за лимит 28ч
- 🚨 Customer Success при 50+ юзерах в 3+ вертикалях

**Итог:** стратегия Y сохраняется, но **правильная причина**: ограничивают
маркетинг и экспертиза, не код. Оптимум = 2 вертикали до M18.

**Внутри Y выбран подвариант Y(3) «holding»:**
- SecAudit.ru узкий бренд для ИБ (не ре-брендируем — сохраняем
  накопленный SEO/узнаваемость)
- Шаблонизатор = узкий бренд для самозанятых на отдельном домене
- **Но архитектура `core/` + `domains/` с первого дня** — позволяет на
  M18+ объявить зонтичный бренд (как Яндекс.Такси + Яндекс.Еда) без
  принудительного ре-брендинга
- Ре-брендинг в Y(2) generic-платформу — только если M18 оба работают +
  retention ≥50% + есть 3-я вертикаль с подтверждённым PMF

Детали: `decisions/multi-vertical-analysis.md` (§ «Пересмотр» + § 4).
